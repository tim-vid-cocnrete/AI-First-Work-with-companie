# Справочник статусов Гибрид
**Описание**: Интерпретация статусов воронки Гибрид/ISA (Work, Offer, Paid, Later).
**Категория**: Marketing

> [!IMPORTANT]
> **Источник**: `Docs/SalesAndMarketing/User Research/hybrid_statuses_interpretation.md`

## 1. Активная воронка (Pipeline)
Этапы, через которые проходит клиент на пути к продаже.

| Статус (CRM код) | Название | Интерпретация |
| :--- | :--- | :--- |
| **Work** | **В работе / Взятие в работу** | Сделка создана, менеджер начал первичную обработку. Стартовый этап воронки. |
| **Qualify** | **Квалификация** | Этап первичного отсева и выявления потребностей (часто объединен с `Work` или предшествует `Offer`). |
| **Offer** | **Предложение / Оффер** | Клиенту сделано конкретное предложение (презентованы условия, стоимость, программа). Ключевой этап перед продажей. |
| **Link** | **Ссылка на оплату / Интерес** | Клиент выразил готовность купить и получил ссылку на оплату. "Горячий" этап (ожидание транзакции). |
| **Credit** | **Рассрочка / 7Pay** | Клиент оформляет рассрочку или кредит (банковский продукт). Указывает на чувствительность к цене. |

## 2. Успех (Success)
Конечный желаемый статус.

| Статус (CRM код) | Название | Интерпретация |
| :--- | :--- | :--- |
| **Paid** | **Оплатил / Успешная оплата** | Сделка успешно закрыта, деньги поступили (или оформлена ISA/рассрочка). Клиент стал студентом. |

## 3. Отложенный спрос (Deferred)
Клиенты, которые не отказались, но взяли паузу.

| Статус (CRM код) | Название | Интерпретация |
| :--- | :--- | :--- |
| **Later** | **Начнет в перспективе** | "Ждуны". Клиент просит перенести старт (нет денег, учеба, отпуск). Резерв для реанимации (Reactivated). |

## 4. Отказ и Потери (Failed / Lost)
Причины срыва сделки.

| Статус (CRM код) | Название | Интерпретация |
| :--- | :--- | :--- |
| **NoCall** | **Недозвон** | До клиента не удалось дозвониться ни разу. "Мертвый" лид с самого начала. |
| **Cancel** | **Отказ** | Явный отказ клиента (дорого, нет времени, передумал, конкуренты). |
