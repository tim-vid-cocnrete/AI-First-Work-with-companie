# Стадии Лидов и Сделок

> [!IMPORTANT]
> **Источник**: `Docs/SalesAndMarketing/Process/lead_deal_stages.md`

## 1. Правила работы с Лидами

### Статус: "Не обработан"
*   **SLA:** Прозвон в течение **15 минут**.
*   **Действие:** Первый контакт. Если иностранный номер — передать чат-менеджеру.

### Статус: "После дубля"
*   **SLA:** 15 минут.
*   **Суть:** Повторная заявка. Если старый лид закрыт <14 дней назад — работаем в старом. Если >14 дней — создаем новый.

### Статус: "2 звонок / 3 звонок"
*   Если не дозвонились с 1 раза -> **2 звонок** (авто-письмо + смс). Звоним через 1 час.
*   Если не дозвонились со 2 раза -> **3 звонок**. Звоним на следующий день.
*   Итого в первый день минимум 3 звонка.

### Статус: "НО (Не отвечает) День 1 / День 2 / День 3"
*   Цикл дозвона для молчунов. Звоним с разных номеров.
*   Если за 3 дня тишина -> **Тотальный недозвон**.

### Классификация отказов (Закрытие лида)
*   **Не подходит:** Возраст, Гражданство (не РФ для постоплаты), Банкрот, Долги, Судимость.
*   **В перспективе:** Готов начать через 1-3 месяца.
*   **Выбрал конкурентов:** Отработали возражения, но ушел.
*   **Дорого / Нет денег:** Не дали кредит/рассрочку.

## 2. Смарт-процесс "Гибрид Продажи"

### Воронка "Оплата"
*   **Назначен ответственный:** Звонок в течение 1 часа.
*   **Недозвон после квалификации:** 6 звонков за 4 дня.
*   **Взят в работу:** Активная фаза переговоров.
*   **Заявка на рассрочку / Ссылка на оплату:** Клиент в процессе оплаты. Срок нахождения: 3 дня.
*   **Оплатил:** Финальный статус успеха.

### Воронка "Заключение договора"
1.  **Подготовка договора:** Ждем паспорт.
2.  **Согласование:** Клиент читает.
3.  **Делает ЭЦП:** Клиент оформляет подпись.
4.  **Отправлен на подписание:** В Диадоке.
5.  **Договор подписан.**

### Воронка "Формирование потока"
*   Ручное распределение оплативших студентов по потокам (Java, FE, Go, QA).
*   Перенос в "Начал обучение" после старта.
