## 3. Бизнес-Проекты (Стажировка)

**Цель:** Опыт командной разработки, подтверждение навыков на практике.

### Java Project
*   **Команда:** 6-7 человек.
*   **Срок:** 4-6 недель.
*   **Процесс:** Тимлид-ментор выдает задачи и проводит финальное ревью. Студенты работают с Git (GitHub, GitLab, Bitbucket), тестируют код, делают код-ревью.
*   **Дейлики:** 2-3 раза в неделю созвоны с тимлидом.
*   **Инструменты:** Jira.
*   **Примеры задач:**
    *   Настроить аспект логирования.
    *   Написать модульные тесты.
    *   Идентифицировать и оптимизировать узкие места.
    *   CI/CD.
    *   Swagger/OpenApi.

### Frontend Project
*   **Команда:** 5-6 человек.
*   **Срок:** 4-6 недель.
*   **Примеры задач:**
    *   Привести в порядок стили Header.
    *   Разработать компонент карточки товара.
    *   Разработать страницу карточки магазина.
    *   Адаптировать шапку под мобильные.

### Golang Project
*   **Команда:** 7-8 человек.
*   **Срок:** 4-6 недель.
*   **Примеры задач:**
    *   Написать сервис используя библиотеки golight.
    *   Сгенерировать слой service для CRUD операций.
    *   Миграции (goose).
    *   Graceful shutdown.
    *   Уведомления по email.
    *   Helm чарты для метрик.

### QA Project
*   **Команда:** 3-4 человека.
*   **Срок:** 2-3 недели.
*   **Задачи:** Пишут тест-кейсы, проверяют задачи, ведут документацию.
*   **Примеры задач:**
    *   Протестировать DELETE /passengers/{id}.
    *   Протестировать GET /flights/status.
    *   Протестировать POST /flights.

## 5. Процесс Обучения
1.  **Старт:** Организационный колл (знакомство).
2.  **Теория:** Изучение материалов на платформе.
3.  **Практика:** Отработка навыков, детальный фидбэк от ментора.
4.  **Комьюнити:** Обучение в группе, поддержка.
5.  **Менторство:** Еженедельные встречи (Q&A).
6.  **Ревью:** Проверка материала 1-на-1 или в группах.
7.  **Проект:** Командная стажировка (4-6 недель) с тимлидом.
